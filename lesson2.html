The first program everyone writes:

Lines 1 and 5 define a class

A class lets you bundle related functionality into one file, and use it in another

All functionality must be defined inside a class, only package and import go outside.


						public class HelloWorld {
						  public static void main(String[] args) {
						    System.out.println("Hello, world!");
						  }
						}

Lines 2 and 4 define the main method

The main method is the starting point of every Java program, and must be written like this

When the program is run, whatever is inside the main runs.


						public class HelloWorld {
						  public static void main(String[] args) {
						    System.out.println("Hello, world!");
						  }
						}

Line 3 is a method call (using a method) on System, in this case to output a piece of text (a String) to the terminal/console.

Bonus slide, the main method 

						public static void main(String[] args) {
						  // Your code here
						}

The main method is made of a few key words:

  • public is an access modifier - it says who can see and use this. Everyone can see public definitions.
  • static means only one of the defined things will ever exist, and would be shared between whoever called it.
    This means it exists and is usable immediately (very handy for Java).
  • void is the return type - it defines what sort of thing you get back if you use this method in your program. Void however means that nothing is returned.
Variables

As we've seen before, variables are for storing values we can refer to and update in our program.

We've been putting whatever we want into our pseudocode variables, Java is more picky.

Java is strictly typed - meaning you must provide the type of the variable and stick to it

This is the general pattern:


							ObjectType nameOfObject = value;

Here are some concrete examples:


						public class VariablesAndTypes {
						  public static void main(String[] args) {
						    String greeting = "Hello";
						    Integer myFavouriteNumber = 17;
						    Boolean isThisJava = true;
						  }
						}
Types

As mentioned before, Java is strictly typed

meaning you need to tell and stick to the type you give things.

Java provides the fundamental types with which people can (and have) built everything

Among those are basic data types that we are concerned with:

  • Strings (type String)
  • Numbers (types Integer and Double)
  • Booleans or true/false values (type Boolean)
String

Represents text in your program, they must be enclosed in quote marks ("")


						public class UsingStrings {
						  public static void main(String[] args) {
							String teacherName = "Ben";
							String courseName = "Java for beginners";
							String statement = teacherName + " teaches " + courseName;
							System.out.println(statement);
						  }
						}

Notice that we can use + to append/glue Strings to each other.

This works for other types too, Java does the conversion to String for us.

Numbers

Java needs you to choose between a number that:

  • Only does whole numbers (type Integer or Long)
  • Has a decimal place (type Float or Double)
  • Handles even larger numbers (at the cost of memory)

						public class UsingNumbers {
						  public static void main(String[] args) {
						    Integer i1 = 123;
						    Long l1 = 123l;
						    Float f1 = 123.45f;
						    Double d1 = 123.45;
						  }
						}

Operators

Mathematical Operators

Along with the fundamental number types, Java gives us convenient operators to do math:


						public class MathOperators {
						  public static void main(String[] args) {
						   Integer plus = 1 + 2;
						    Integer minus = 1 - 2;
						    Integer divide = 1 / 2;
						    Integer multiply = 1 * 2;
						    Double power = Math.pow(1, 2); // 1 to the power of 2
						    Integer modulus = 1 % 2; // remainder of 1 divided by 2
						  }
						}

Note modulus (%) - which is the remainder of integer division, and is more useful than you think it is.

These operators follow BODMAS, you can use brackets as normal to control the order.

Comparison Operators

In addition to mathematical calculators, Java also provides handy number comparison:


							public class MathComparisons {
							  public static void main(String[] args) {
							    Integer lessThan = 1 < 2;
							    Integer greaterThan = 1 > 2;
							    Integer lessThanOrEqual = 1 <= 2;
							    Integer greaterThanOrEqual = 1 >= 2;
							    Double equal = 1 == 2; // note it is two equals
							  }
							}
Boolean Operators

Boolean Operators

Java also provides operators for booleans:


							public class BooleanOperators {
							  public static void main(String[] args) {
							    Boolean inclusiveOr = true || false;
							    Boolean and = true && false;
							    Boolean not = !true;
							  }
							}

if-statements

Another familiar tool, give us conditional execution in our code.


							public class IfStatements {
							  public static void main(String[] args) {
							    Boolean condition = true;

							    if (condition) {
							      System.out.println("The condition is true");
							    }

							    if (condition) {
							      System.out.println("The condition is true");
							    } else {
							      System.out.println("The condition is false");
							    }
							  }
							}

In these examples we're using a Boolean variable, you can however inline any expression that returns a Boolean.

This example compares


							public class IfStatementsContinued {

							    Integer numberToCheck = 5;

							    if (numberToCheck < 0) {
							      System.out.println("The number is too small");
							    } else if (numberToCheck > 100){
							      System.out.println("The number is too large");
							    } else {
							       System.out.println("The number is between 0 and 100");
							    }
							  }
							}
